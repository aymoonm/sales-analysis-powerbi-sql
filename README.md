# Sales Data Analysis using SQL Server & Power BI

## 📌 Project Overview

This project demonstrates an end-to-end sales data analysis workflow using **SQL Server** and **Power BI**. The dataset contains sales transactions from six countries: **Canada, China, India, Nigeria, the United Kingdom, and the United States**.

The objective of the project was to integrate multiple datasets, clean and prepare the data, perform business analysis using SQL, and build an interactive Power BI dashboard to support data-driven decision-making.

---

## 🛠️ Tools & Technologies

* **SQL Server** – Data integration, cleaning, transformation, and business analysis.
* **Power BI Desktop** – Data modeling, DAX calculations, and interactive dashboard development.

---

## 📂 Project Workflow

### 1. Data Integration

* Combined six country-specific datasets into a single sales table using `UNION ALL`.
* Standardized the dataset structure for analysis.

### 2. Data Cleaning

* Checked for missing and inconsistent values.
* Corrected invalid records.
* Updated missing values where appropriate.
* Validated data quality before analysis.

### 3. SQL Analysis

Performed business analysis using SQL queries, including:

* Revenue, cost, and profit analysis.
* Top-selling products.
* Best-performing sales representatives.
* Sales performance by country and store.
* Customer demographic analysis.
* Sales trend analysis.

### 4. Dashboard Development

Built an interactive Power BI dashboard featuring:

* KPI Cards
* Sales Trend Analysis
* Revenue and Profit Breakdown
* Country Performance
* Product Performance
* Customer Demographics
* Interactive Filters and Slicers

---

## 📊 Dashboard Features

The dashboard allows users to:

* Monitor overall sales performance.
* Compare revenue and profit across countries.
* Identify top-performing products.
* Evaluate sales representative performance.
* Analyze customer demographics.
* Explore data dynamically using filters.
